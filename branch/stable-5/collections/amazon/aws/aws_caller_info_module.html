amazon.aws.aws_caller_info module – Get information about the user and account being used to make AWS calls

Note

This module is part of the amazon.aws collection (version 5.1.0).

To install it, use: ansible-galaxy collection install amazon.aws. You need further requirements to be able to use this module, see Requirements for details.

To use it in a playbook, specify: amazon.aws.aws_caller_info.

New in amazon.aws 1.0.0

  • Synopsis

  • Requirements

  • Parameters

  • Notes

  • Examples

  • Return Values

Synopsis

  • This module returns information about the account and user / role from which the AWS access tokens originate.

  • The primary use of this is to get the account id for templating into ARNs or similar to avoid needing to specify this information in inventory.

Requirements

The below requirements are needed on the host that executes this module.

  • python >= 3.6

  • boto3 >= 1.18.0

  • botocore >= 1.21.0

Return Values

Common return values are documented here, the following are the fields unique to this module:

Key

Description

account

string

The account id the access credentials are associated with.

Returned: success

Sample: "123456789012"

account_alias

string

The account alias the access credentials are associated with.

Returned: when caller has the iam:ListAccountAliases permission

Sample: "acme-production"

arn

string

The arn identifying the user the credentials are associated with.

Returned: success

Sample: "arn:aws:sts::123456789012:federated-user/my-federated-user-name"

user_id

string

The user id the access credentials are associated with. Note that this may not correspond to anything you can look up in the case of roles or federated identities.

Returned: success

Sample: "123456789012:my-federated-user-name"
